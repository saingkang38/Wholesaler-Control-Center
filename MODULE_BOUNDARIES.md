# 모듈 경계 정의 문서
버전: 1.0
상태: 운영 기준 확정본

## 문서 목적
이 문서는 각 모듈의 책임과 비책임을 정의한다.  
변경 요청이 들어왔을 때 어디를 먼저 수정해야 하는지 판단하는 기준으로 사용한다.

목표는 하나다.  
**기능이 늘어나도 구조가 쉽게 꼬이지 않게 만드는 것.**

## 2. auth

### 책임
- 로그인
- 로그아웃
- 세션 처리
- 인증 검증

### 책임 아님
- 사용자 목록 관리
- 도매처 수집
- 비교 판정
- 파일 출력 생성

---

## 3. users

### 책임
- 사용자 생성/수정/비활성화
- 역할 관리
- 계정 상태 관리

### 책임 아님
- 로그인 세션 처리
- 도매처 계정 로그인
- 수집 실행

---

## 4. dashboard

### 책임
- 요약 정보 표시
- 최근 실행 결과 표시
- 빠른 실행 버튼 연결
- 최근 다운로드/비교 결과 표시

### 책임 아님
- 실제 수집 실행 로직
- 실제 비교 로직
- 실제 exporter 로직

### 원칙
dashboard는 보여주고 연결만 한다.  
실제 작업은 service/orchestration 계층이 담당한다.

---

## 5. wholesalers

### 책임
- 도매처 기본 정보 관리
- 도매처 활성/비활성
- 도매처 메타 정보 관리

### 책임 아님
- 실제 로그인
- 실제 수집
- 비교 결과 생성

---

## 6. collectors

### 책임
- 도매처 사이트 로그인
- 목록 수집
- 상세 수집
- raw 데이터 반환
- 도매처별 수집 로그 생성

### 책임 아님
- 사용자 인증
- 비교 판정
- 커스텀 출력 생성
- 대시보드 표시

### 원칙
1. 도매처별 collector는 서로 독립적이어야 한다.
2. ownerclan 문제는 ownerclan collector 내부에서 우선 해결한다.
3. 특정 도매처 예외를 공통 collector 파일에 누적하지 않는다.
4. collector는 raw 수집까지만 책임지고, compare/export로 넘어가지 않는다.

---

## 7. normalization

### 책임
- raw 수집 데이터를 공통 상품 구조로 변환
- 상태값 통일
- 공통 컬럼 매핑
- 기본 정리/정규화 처리

### 책임 아님
- 실제 사이트 접속
- 비교 판정
- 출력 포맷 생성

### 원칙
수집(raw)과 정규화(normalized)는 반드시 분리한다.

---

## 8. products

### 책임
- 정규화된 상품 데이터 저장/조회
- 상품 검색/필터
- 상품 상태 목록 제공

### 책임 아님
- 수집 실행
- 비교 실행
- 출력 포맷 변환

---

## 9. compare

### 책임
- 기준 파일과 현재 상품 데이터 비교
- 상태 판정
- diff 생성
- 비교 결과 저장

### 책임 아님
- 도매처 로그인/수집
- 사용자 로그인
- 파일 물리 저장
- 다운로드 파일 생성

### 원칙
1. 비교 기준 변경은 compare 내부에서 우선 해결한다.
2. 비교 규칙을 UI 코드에 넣지 않는다.
3. 규칙이 늘어나면 rule 계층으로 분리한다.

---

## 10. exporters

### 책임
- 통합 마스터 파일 생성
- 비교 결과 파일 생성
- 커스텀 양식 파일 생성

### 책임 아님
- 상품 수집
- 비교 판정
- 사용자 인증

### 원칙
1. 포맷별 exporter를 분리한다.
2. 포맷 분기문을 한 파일에 누적하지 않는다.
3. 복잡한 출력 규칙은 exporter 내부로 모은다.

---

## 11. scheduler

### 책임
- 예약 작업 등록
- 매일 1회 자동 실행 호출
- 자동 실행 결과 연결

### 책임 아님
- collector 세부 구현
- compare 규칙 구현
- exporter 포맷 구현

### 원칙
자동 실행은 호출만 담당하고, 실제 수집/비교/출력 세부 로직은 해당 모듈이 담당한다.

---

### 책임 아님

## 13. files

- 다운로드 파일 경로 관리
- 파일 메타데이터 관리

### 책임 아님
- 비교 판정
- exporter 내부 포맷 변환
- 수집 로직

- 기본 옵션
- 향후 설정화 가능한 규칙 관리

### 책임 아님

- 수집 실행 본체

---

## 15. infrastructure

### 책임
- DB 연결
- 환경변수 로드
- 브라우저 자동화 도구 연결
- 외부 API 연결
- 파일 스토리지 연결

### 책임 아님
- 비즈니스 규칙 정의
- 비교 판정 규칙
- 도매처 업무 정책

---

## 16. service / orchestration 경계

### route / controller
- 요청 수신
- 입력 검증
- service 호출
- 응답 반환

### service / orchestration
- 실행 흐름 제어
- 여러 모듈 조합
- run 생성 / 상태 변경 / 결과 연결

### module 내부 로직
- 자기 책임 영역의 실제 처리

### 금지
- route에서 collector 세부 구현 직접 처리
- dashboard에서 compare 규칙 직접 처리
- exporter에서 수집 로직 직접 처리
- collector에서 UI 응답 포맷까지 직접 결정

---

## 17. 수정 시 판단 기준

### 도매처 로그인/수집 문제
- 먼저 `collectors/{wholesaler_name}/` 확인

### 상태 판정/비교 문제
- 먼저 `compare/` 확인

### 출력 양식 문제
- 먼저 `exporters/{format_name}/` 확인

### 로그인/권한 문제
- 먼저 `auth/`, `users/` 확인

### 자동 실행 문제
- 먼저 `scheduler/` 또는 execution orchestration 확인

### 설정값 문제
- 먼저 `settings/` 또는 `CONFIG_POLICY.md` 확인

---

## 18. 절대 금지

1. collector 예외를 공통 서비스 파일에 누적
2. compare 규칙을 UI 코드에 배치
3. exporter 포맷 처리를 controller에서 직접 구현
4. auth 로직을 collector에 혼합
5. 하나의 파일이 여러 모듈 책임을 동시에 가짐
6. 임시 대응을 이유로 구조 경계를 무너뜨림

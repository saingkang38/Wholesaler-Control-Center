# 검증지침

코드 수정 후 반드시 해당 유형의 체크리스트를 수행한다.

---

## Flask/서버 코드 변경

- [ ] 서버 재시작 (`python run.py`)
- [ ] `localhost:5000` 응답 확인 (302 또는 200)
- [ ] 서버 로그에 ERROR 없음 확인

---

## 시그널 감지 로직 변경

대상 파일: `app/actions/__init__.py` (detect 영역)

- [ ] 대시보드 → 수동으로 도매처 1곳 수집 실행
- [ ] DB `action_signals` 신규 생성 여부 확인
- [ ] 의도한 시그널만 생성되고 불필요한 시그널 없음 확인

---

## 액션 실행 로직 변경

대상 파일: `app/actions/__init__.py` (execute 영역)

- [ ] 시그널 하나 승인(테스트용)
- [ ] `action_signals.status = executed` 전환 확인
- [ ] `error_message = NULL` 확인
- [ ] 네이버 스토어에서 실제 반영 확인

---

## 수집기(collector) 변경

대상 파일: `app/collectors/{도매처코드}/`

- [ ] 해당 도매처 수동 수집 실행
- [ ] `collection_runs.status = success` 확인
- [ ] 수집 건수가 0이 아님 확인

---

## DB 모델/스키마 변경

- [ ] 앱 재시작 후 기존 데이터 정상 조회 확인
- [ ] 신규 컬럼이 있으면 기존 행의 기본값 확인
- [ ] 관련 페이지 접속 시 500 에러 없음 확인

---

## 템플릿/UI 변경

대상 파일: `templates/`

- [ ] 브라우저에서 해당 페이지 직접 접속
- [ ] 데이터 표시 정상 확인
- [ ] 브라우저 콘솔 에러 없음 확인

---

## 스케줄러 변경

대상 파일: `scheduler.py`

- [ ] `scheduler.py` 재시작
- [ ] 로그에 "스케줄러 시작" 메시지 확인
- [ ] 등록된 job 목록과 다음 실행 시간 로그 확인

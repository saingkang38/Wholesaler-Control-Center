# 도매처 통합 관리 시스템 PRD
버전: 1.0
상태: 운영 기준 확정본

---

## 1. 프로젝트 목적

여러 도매처 사이트의 상품 데이터를 수집하여 하나의 통합 데이터로 관리하고,  
기준 파일과 비교하여 상태 변화를 판별한 뒤,  
업무에 필요한 양식으로 다운로드할 수 있는 웹 기반 관리 시스템을 구축한다.

이 프로젝트는 아래 목적을 동시에 만족해야 한다.

1. 프로그램을 실제로 완성한다.
2. 토큰 사용량을 최소화하는 방식으로 개발한다.
3. 비개발자가 단계별로 따라갈 수 있게 만든다.
4. 차후 유지보수와 모듈 확장이 쉽도록 설계한다.
5. 가능한 범위에서 자동화를 최대화한다.
6. Claude Code 관련 운영 원칙은 공식 배포 문서 기준만 반영하고, 베타/추측성 정보는 배제한다.

---

## 2. 해결하려는 문제

현재 업무에는 아래 문제가 있다.

- 도매처별 사이트 구조가 달라 수집 작업이 비효율적임
- 수집 결과를 한 곳에서 관리하기 어려움
- 기준 파일과 현재 데이터를 비교하는 작업이 느리고 불안정함
- 품절/단종/재입고/수정 여부를 빠르게 확인하기 어려움
- 출력 양식이 채널별로 달라 반복 작업이 큼
- 사람이 직접 반복 확인해야 하는 작업이 많음
- 나중에 도매처가 늘어나면 구조가 쉽게 꼬일 가능성이 큼

---

## 3. 핵심 목표

### 3-1. 비즈니스 목표
- 상품 수집 시간을 줄인다.
- 상태 변경 대응 속도를 높인다.
- 직원과 동일 시스템을 공유한다.
- 결과 데이터를 업로드용 양식으로 빠르게 전환한다.
- 도매처가 늘어나도 유지 가능한 구조를 만든다.
- 수작업을 줄이고 자동화를 늘린다.

### 3-2. 시스템 목표
- 도매처별 collector 분리
- 로그인 기반 웹 관리 시스템
- 통합 DB 기반 운영
- 비교 엔진과 출력 엔진 분리
- 자동 실행과 수동 실행 지원
- 여러 장소에서 접속 가능한 구조
- 구조 변경 없이 새 도매처와 새 출력 양식을 추가 가능한 구조

### 3-3. 개발 운영 목표
- 비개발자도 단계별로 진행 상황을 이해할 수 있어야 한다.
- Claude Code는 한 번에 작은 범위만 작업해야 한다.
- 토큰 낭비를 막기 위해 작업 단위를 잘게 유지해야 한다.
- 설명보다 실행 결과와 확인 절차를 우선해야 한다.

## 8. MVP 범위

1차 MVP는 아래를 만족하면 된다.

1. 관리자/직원 로그인 가능
2. 오너클랜 1개 도매처 수집 가능
3. 수집 데이터를 DB에 저장 가능
4. 통합 상품 목록 조회 가능
5. 매일 1회 자동 실행 가능
6. 통합 마스터 파일 다운로드 가능
7. 기준 파일 업로드 가능
8. 비교 결과 생성 가능
9. 비교 결과 다운로드 가능
10. 실패 원인 로그 추적 가능

## 13. 개발 방식 원칙

1. 한 번에 전체를 만들지 않는다.
2. 단계별로 작게 나눠서 진행한다.
3. 현재 단계 완료 전 다음 단계 핵심 구현을 하지 않는다.
4. 비개발자 실행 항목과 Claude 작업 항목을 분리한다.
5. 설명보다 결과물과 확인 방법을 우선한다.
6. 토큰 절약을 위해 요청 범위 밖 탐색을 줄인다.
7. 공식 배포 문서로 확인된 내용만 반영한다.

---

## 14. Claude Code 적용 원칙

이 프로젝트에서 Claude Code 관련 작업은 아래 원칙을 따른다.

1. 공식 배포 문서 기준만 반영한다.
2. 베타 기능, 실험 기능, 커뮤니티 추측, 불확실한 팁은 반영하지 않는다.
3. 요청 범위 밖 파일 전체 재탐색을 기본값으로 하지 않는다.
4. 한 번에 한 작업 단위만 처리한다.
5. 비개발자가 바로 실행할 수 있는 형태로 답한다.
6. 긴 설명으로 토큰을 낭비하지 않는다.
7. 문서와 범위를 먼저 보고 코드 작업을 진행한다.
